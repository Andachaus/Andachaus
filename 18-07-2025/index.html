Camelcase

Lo primero que recuerdo es en torno al camelcase, una técnica de escritura para tener claridad en nombres.
La he usado para nombrar objetos, archivos y variables de tal forma que el nombre puede contener varias palabras pero sin espacios, además, la primera palabra es en minúsculas y las otras palabras van com mayusculas sostenida.

Lo ideal es crear el proyecto en una carpeta, donde se pueda alojar tanto el archivo html como otros archivos complementarios.Además, es necesario poder revisar siempre que el código esté bien, para eso lo mejor es usar correcto en linea como puede ser Validator HTML

Los 3 importantes

Un buen código de página web debe tener siempre estos 3 aspectos:

  1. Doctype
  2. Head
  3. Body

Doctype

Este es un tipo de protocolo que determina que tipo de archivo va a ser, en este caso se pone html
se escribe !DOCTYPE html

Head

Aquí es donde va metadata, como el tipo de carácteres (UTF-8 o algo así), el titulo que aparece en la pestaña del navegador y la miniatura del navegador.

Body

Aquí es donde iría el grueso de la página web, donde va a estar todo lo visible para el usuario, aquí estará el contenido de la página.

Etiquetas y Atributos

  • Estas definen la estructura y tipo de código, están entre estas llaves: < />
  • Las normales inician y terminan, ej: <p> <p />
  • Otras son autoconcluyentes, ej: <img >
  • Algunas etiquetas tienen atributos, que son información adicional que va dentro de la etiqueta correspondiente
  • Hay simbolos que se usan para la estructura del codigo html como lo son el < o >, para usar estos como texto normal, es necesario llamarlos con otros nombres (& l t)
  • Parrafo <p> <p/>
  • Salto de linea <br> (auto concluyente)
  • No usar H1 o H2 por el estilo, hay que tener en cuenta la sintaxis y la semántica (en mis palabras sería tener en cuenta la jerarquía de cada etiqueta)
  • Existen listas ordenadas y desordenadas, y estas creo que tienen sus propios atributos
